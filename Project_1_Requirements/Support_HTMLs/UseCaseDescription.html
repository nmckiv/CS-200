ChocAn Project 1 - Requirements

Use Case Descriptions

Request Provider Directory

Use Case: Request Provider Directory
Context: The provider requests the provider directory to be sent as an attachment to their e-mail inbox
Actors: Provider
Main Success Scenario:
1. The provider requests the provider directory.
2. The provider enters the e-mail address wwhere they want the directory sent.
3. The provider directory is sent to the given e-mail address as an attachment.
4. Use case is finished
Extensions: N/A

Bill ChocAn

Use Case: Bill ChocAn
Context: Provider is recording a service and billing ChocAn the proper amount
Actors: Provider
Main Success Scenario:
1. Provider enters login information
2. Provider is verified through the ChocAn database
3. Provider enters the member info to have it verified
4. Provider records that the desired service was completed
4.1. Optionally the Provider adds comments to the record
5. The provider verifies that the fee for the service is correct
6. The provider fills out their own personal record for data keeping purposes
7. Use case is finished
Extensions:
1.a. Invalid provider login
1.a.1. System notifies the provider
1.a.2. Use case returns to step 1
1.a.2.a. After 3 invalid attempts, option to contact customer service is offered then proceed to step 7
3.a. Invalid member information
3.a.1 Use case returns to step 3

Verify User

Use Case: Verify User
Context: A user is verified through the ChocAn System
Actors: Provider, Manager, ChocAn Operator
Main Success Scenario:
1. Provider logins in to provider system
2. Inputs user info
3. User is verified within the system
4. Validated appears on the one-line display
5. Use Case is finished
Extensions:
1.a. Manger logins in to manager system
1.b. ChocAn Operator logins in to operator system
4`.a. Invalid appears on the one-line display

Validate Member

Use Case: Validate Member
Context: Provider validates a member's information to confirm they are part of ChocAn
Actors: Provider
Main Success Scenario:
1. Software prompts provider to enter member information
2. Provider enters member's info or swipes the member's card
3. ChocAn returns whether or not the member is a part of ChocAn
4. Use case is finished
Extensions: N/A

Run Main Accounting Procedure

Use Case: Run Main Accounting Procedure
Context: The timer is automated to make and distribute reports on the end of Friday at midnight that detail the weeks activity
Actors: Timer
Main Success Scenario:
1. The timer is prompted when it becomes Midnight on Friday
2. The reports are created
3. The reports are sent out as email attachments
4. Use case is finished
Extensions:
2.a. Member report
2.a.1. The system will look up each member who used a ChocAn provider during the last week
2.a.2. For each member the system will access the member's name, street address, cirt, state, ZIP code, and their member number. The system will also find what services each member was provided during the week and record the date of service, provider name, and service name
2.a.3. Use case progresses to step 3
2.b. Provider report
2.b.1. The system looks up each provider who has billed ChocAn during the previous week
2.b.2. For each provider who billed ChocAn, the system locates the provider's name, number, street address, city, state, and ZIP code. Then, for each service provided the system will look up the date of service, the date and time the record of service was recieved by ChocAn, member name, member number, service code, and the fee to be paid
2.b.3. The system totals the number of consultations for each provider during the week andthe total fee that needs to be paid to the provider (up to $99,999,999)
2.b.4. Use case progresses to step 3
2.c. Manager report
2.c.1. The system looks up what providers have been used for the week
2.c.2. For each provider, the system will find how many times each provider was used and the total fees owed to the provider
2.c.3 The system records the total number of providers and the total number of consultations with members during that week
2.c.4. Use case progresses to step 3
2.d. EFT report
2.d.1. A record of EFT data is written to a disk
2.d.2 Use case progresses to step 3

Modify Provider

Use Case: Modify Provider
Context: ChocAn Operator edits ChocAn provider status
Actors: ChocAn Operator
Main Success Scenario:
1. Operator logs into system
2. Operator is shown Operator menu
3. Operator selects to view records updated by ACME
4. Operator edits provider database
5. Use case is finished
Extensions:
1.a. Invalid Operator Credentials
1.a.1. System notifies user
1.a.2. Use case returns to step 1
1.a.3. After 5 failed attempts, use case progresses to step 6
4.a. Operator adds a new provider
4.a.1. Operator selects new provider option
4.a.2. Operator inputs new provider's information to ChocAn database
4.a.3 New provider is assigned an ID
4.a.4 New provider account is created
4.a.5 Use case returns to step 2
4.b. Operator removes a provider
4.b.1. Operator selects remove provider option
4.b.2. Operator deletes provider account
4.c.3 Use case returns to step 2
4.c. Operator updates provider information
4.c.1. Operator selects update provider option
4.c.2. Operator updates provider account
4.c.3 Use case returns to step 2

Generate Individual Report

Use Case: Generate Individual Report
Context: At any time a manager can request an individual report of a service that was completed
Actors: Manager
Main Success Scenario:
1. Manager logs into system
2. Manager is shown report request menu
3. Database fetches report and returns it to the manager
4. Use case is finished
Extensions:
1.a. Invalid Manager Credentials
1.a.1. System notifies user
1.a.2 Use case return to step 1
1.a.3 After 3 failed attemps, use case progresses to step 4
3.a. Manager requests a member report
3.a.1. Manager selects the desired member's report
3.a.2. Member report is returned to the manager
3.a.3. Use case progresses to step 4
3.b. Manager requests a manager report
3.b.1. Manager selects the desired manager's report
3.b.2. Manager report is returned to the manager
3.b.3. Use case progresses to step 4
3.c. Manager requests the EFT report
3.c.1. The EFT report is returned to the manager
3.c.2. Use case progresses to step 4
3.d. Manager requests a provider report
3.d.1. Manager selects the desired provider's report
3.d.2. Provider report is returned to the manager
3.d.3. Use case progresses to step 4
